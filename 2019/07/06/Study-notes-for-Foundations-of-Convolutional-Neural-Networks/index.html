Foundations of Convolutional Neural Networks

Computer vision

Image classification

Object detection

Neural style transfer

etc.

Dimension of input features can get really big.

e.g. A picture of 1000 * 1000 has 3 million input features. (1000 * 1000 * 3)


Edge detection example

In mathematics, ‘*‘ denotes convolution. In python, ‘*‘ also denotes matrix wise multiplication.

Image * filter = new image

Python: conv_forward()

Tensorflow: tf.nn.Conv2D()

Keras: Conv2D()

A vertical edge is a 3 x 3 region where there’re bright pixels on the left and dark pixels on the right.


More edge detection

Horizontal:

relatively bright on top, relatively dark below

There’re also other kinds of filters such as Sobel filter, Schorr filter, etc.

You can treat the 9 numbers in the filter matrix as parameters that can learn, instead of fixed, hand-coded values such as the numbers in Sobel filter.


Padding

If you have an n x n original image, a f x f filter, then your output will be (n - f + 1) x (n - f + 1)

But you don’t want it to shrink, or throw away information of the edges of the image.

To solve it, you want to pad the image.

for example, if p = 1, n = 6, f = 3

the dimensions of the original image therefore becomes (6 + 1 + 1) x (6 + 1 + 1) = 8 x 8

after convolution, it becomes 6 x 6, which is the same as that of the original image

“Valid Convolution”:

No padding

n x n * f x f gives you (n - f + 1) x (n - f + 1)

“Same Convolution”:

Output size = Input size

n x n * f x f gives you (n + 2p - f + 1) x (n + 2p - f + 1)

n + 2p - f + 1 = n

p = (f - 1) / 2

f is usually odd so that the filter has a central position and avoids asymmetric padding


Strided convolutions

If stride = 2, for example,

the filter would jump over by 2 steps instead of 1 pixel

Generally:

n x n * f x f, with padding p and stride s

the output dimension would be [ (n + 2p -f) / s + 1 ] by [ (n + 2p -f) / s + 1 ) ]

[z] denotes floor(z), in case (n + 2p - f) isn’t an integer.

You don’t want the filter to interfere with regions outside of the image or image + padding

Technical note on cross-correlation and convolution:

In math textbooks, you mirror the filter both on vertical and horizontal axis, before doing the multiplication.

By convention, in DL, we don’t bother to do the step of flipping it over. What we call “convolution” in DL is actually “cross-correlation” in mathematics, but by convention we just call it “convolution” and it works just fine.


Convolutions over volume

Convolutions on RGB image

Height x Width x # Channels

The # Channels(or sometimes also called depth) of the image and the filter must be the same.

It’s potentially possible to have a filter that cares about only one of red, blue, and green.

What if we want to detect edges of both horizontal and vertical?

Convolve with 2 filters, resulting in two outputs and then stack them up.

In general:

n x n x nc * f x f x nc = (n - f + 1) x (n - f + 1) x nc’, assuming no padding and no stride

nc’ denotes the number of filters you use.


One layer of a Convolutional Network

Recall that

Z[1] = W[1] * X + b[1]

A[1] = g[1](Z[1])

In CNN

Filter plays a role similar to W[1]

You would also add a bias term(a real number) to the output image, which plays a role similar to b[1]


Number of parameters in one layer:

You have fixed number of parameters regardless of the size of your input.

e.g. If you have 10 filters in size of 3 x 3 x 3, you will have

(3 * 3 * 3 + 1) * 10 = 280 parameters


Summary of notation:

If layer l is a convolution layer

f[l] = filter size

p[l] = padding

s[l] = stride

nc[l] = number of filters

Each filter is : f[l] x f[l] x nc[l - 1]

Activations: a[l] = nH[l] x nW[l] x nc[l]

If we have m training examples, then activations will be:

A[l] = m x nH[l] x nW[l] x nc[l]

Weights: f[l] x f[l] x nc[l - 1] x nc[l] // nc[l] denotes # filters in layer l

bias: 1 x 1 x 1 x nc[l]

Input: nH[l - 1] x nW[l - 1] x nc[l - 1]

Output: nH[l] x nW[l] x nc[l]


Simple Convolutional Network Example

Take the last activation and flatten it to a big vector.

Fit the vector in a logistic/softmax activation function to get the final yhat.

One general trend is:

As we go deeper, the number of height and width gradually decrease, while the number of channels gradually increase.


Types of Layer in a convolutional network:

  • Convolution (Conv)
  • Pooling (Pool)
  • Fully connected (FC)

Pooling Layers

Max pooling:

Take the max over a f x f region, as if you’re taking a filter size of f.

Hyperparameters:

f(filter size) and s(stride)

f = 2 and s = 2 are common choices.

There’s no parameters to learn though.

performance on each of the channels independently, so applying max pooling won’t change


Average pooling:

Take the average over a f x f region

Max pooling is used much more often than average pooling.

The conv layers tend to have fewer parameters than FC layers, and pooling layers have no parameters.

Why convolutions?

Even though the number of features big, number of parameters of each Conv layer is relatively small.

Parameter sharing:

A feature detector (such as vertical feature detector) that’s useful in one part of the image is also useful in other parts of the image.

Sparsity of connections:

In each layer, each output value depends only on a small number of input.
